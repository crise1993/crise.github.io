# 聊聊js的节流与防抖

防抖和节流都是前端开发中常用的性能优化技术，它们的主要目的都是为了限制某些函数的执行频率，以提高页面性能和用户体验。它们的区别在于触发执行函数的时机不同。

通过本篇文章你能了解到：

  • 什么是节流与防抖
  • 有什么区别，适合怎样的应用场景
  • 如何手写实现节流与防抖

# 节流

当事件被触发后，固定时间间隔内只执行一次该事件。即使在这段时间内事件被触发多次，也只会执行一次。

# 应用场景

适合处理频繁触发的事件。

  • 搜索框输入实时搜索功能：用户在输入过程中，通过防抖可以避免每次输入都发送请求，只有在用户停止输入一段时间后才发送请求。
  • 窗口大小调整：当用户调整窗口大小时，如果触发频率很高，可以通过防抖来减少触发事件，例如调整样式、重新布局等。

# 实现

思路：每次触发事件时都判断当前是否有等待执行的延时函数，没有的话，挂载上；有的话，不挂载

function throttle(fn, wait) {
    let timer = null;

    return function() {
        if(!timer){
            timer = setTimeout(() => {
                fn.apply(this, arguments)
                timer = null
            }, wait);
        }
    }
}

var count = 1;
var container = document.getElementById('container');

function getUserAction(e) {
    container.innerHTML = count++;
};

container.onmousemove = throttle(getUserAction, 1000);

# 防抖

当事件被触发后，一定时间内如果事件没有再次被触发，则执行该事件。如果在这段时间内事件又被触发，则重新计时。

# 应用场景

适合处理高频率的事件。

  • 页面滚动加载：当用户滚动页面时，可以使用节流来限制滚动事件的触发频率，减少滚动时的计算量。
  • 鼠标移动事件：当需要捕获鼠标移动时，可以使用节流来减少事件的触发频率，提高性能。

# 实现

思路：每次触发事件时都判断当前是否有等待执行的延时函数

function debounce(fn, wait) {
    let timer = null;

    return function() {
        // console.log('第1处:', this, arguments)
        clearTimeout(timer);
        timer = setTimeout(() => {
            // 箭头函数没有自己的this和arguments，箭头函数内部的this和arguments取决于父函数
            // console.log('第2处:', this, arguments)
            fn.apply(this, arguments)
        }, wait);
    }
}

var count = 1;
var container = document.getElementById('container');

function getUserAction(e) {
    // console.log('getUserAction:', this, e)
    container.innerHTML = count++;
};

container.onmousemove = debounce(getUserAction, 1000);

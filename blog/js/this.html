# 一文搞定this指向性问题

this关键字是JavaScript中最复杂的机制之一。它是一个很特别的关键字，被自动定义在所有函数的作用域中。

关于this的指向并没有一条万能准则，随着函数使用场合的不同，this的值会发生变化。实际上this是在函数被调用时发生的绑定，它指向什么完全取决于函数在哪里被调用。

下面有几条总结的规则，我们逐条分析下。

  1. 函数属于谁，this就指向谁
  2. 自执行函数中的this指向window
  3. call，apply，bind中this指向被绑定对象
  4. 构造函数中的this指向实例对象
  5. 箭头函数中的this指向同父级函数的this

其中优先级的权重由上至下逐渐加强，当规则发生冲突时，下面的规则优先级更高。 现在我们逐条解释。

# 方法属于谁，this就指向谁

换句话说，就是 函数执行时首先看函数名前面是否有"."，有的话，"."前面是谁，this就指向谁；没有的话this指向window。

# 普通方法的绑定

function fn(){
  console.log(this);
}
var obj={fn:fn};
fn();//this->window
obj.fn();//this->obj
function sum(){
     fn();//this->window
}
sum();
var oo={
 sum:function(){
    console.log(this);//this->oo
    fn()；//this->window
  }
};
oo.sum();

# Dom事件的绑定

oDiv.onclick=function(){
    //this->oDiv
};
oDiv.addEventListener("click",function(){
    //this->oDiv
},false);

至于在IE6~8下使用attachEvent，默认的this指向window，这块我们就不细说了。

# 闭包里面的this指向

只需要记住一点，就是闭包被执行时，闭包属于谁，就指向谁。

this.m=999
var a='a'
var obj={
    m: 100,
    test: function(){
        console.log(this.m, '1')     // 100
        var a = 'b'
        return function(){
            console.log(this.m, '2')     // 999
            return a;
        }
    }
}
obj.test()()  // 'b'
// obj.test()()等同于：
var c = obj.test()
c()  // 'b'

以上的例子中 obj.test() 或 c 实际上是属于window，所以这个🌰中闭包中的this->window

function debounce(fn, wait) {
    let timer = null;

    return function() {
        console.log('第1处:', this)
        clearTimeout(timer);
        timer = setTimeout(function() {
            console.log('第2处:', this)
            // fn.apply(this, arguments)
        }, wait);
    }
}

var count = 1;
var oDiv = document.getElementById('container');

function getUserAction() {
    oDiv.innerHTML = count++;
};

oDiv.onmousemove = debounce(getUserAction, 1000);

以上的例子中，“第1处”的匿名函数属于oDiv，所以 this->oDiv，“第2处”setTimeout属于window，所以setTimeout中匿名回调函数中的 this->window

# 自执行函数中的this指向window

(function(){
    //this->window 
})();
~function(){ 
    //this->window 
}();

# call、apply、bind中this指向被绑定对象

call、apply、bind的作用和实现我们可以查看这篇文章，bind、apply、call。

虽然在call、apply、bind中this指向被绑定对象，但是在严格模式下和非严格模式下，当被绑定对象为undefined、null时，可能得到的值不一样。

var obj={name:"浪里行舟 "};
function fn(num1,num2){
    console.log(num1+num2);
    console.log(this);
}
fn.call(100,200);//this->100 num1=200 num2=undefined
fn.call(obj,100,200);//this->obj num1=100 num2=200

//在非严格模式下
fn.call();//this->window
fn.call(null);//this->window
fn.call(undefined);//this->window
//严格模式下 
fn.call();//在严格模式下this->undefined
fn.call(null);// 在严格模式 下this->null
fn.call(undefined);//在严格模式下this->undefined

apply、bind虽然和call在用法上有所不同，但是this的指向规律完全一致。

# 构造函数中的this指向当前类的实例

function CreateJsPerson(name, age){
    //浏览器默认创建的对象就是我们的实例p1->this
    this.name=name;//->p1.name=name
    this.age=age;
    this.writeJs=function（）{
        // 这里面的this -> 需要看writeJs执行的时候才知道
        console.log("my name is"+this.name +",i can write Js");
   };
//浏览器再把创建的实例默认的进行返回
}
var p1=new CreateJsPerson("尹华芝",48);

但是要注意一点：类中某个方法中的this需要看方法执行的时候，才能确定this指向。

function Fn(){
    this.x=100；//this->f1
    this.getX=function(){
        // this->需要看getX执行的时候才知道
        console.log(this.x);
    }
}
var f1=new Fn;
f1.getX();// 100。方法中的this -> f1
var ss=f1.getX;
ss();// undefined。因为方法中的this -> window

当本条规则和call、apply、bind规则同时出现时，本条规则优先级更高，this->构造函数实例

var value = 2;

var foo = {
    value: 1
};

function bar(name, age) {
    this.habit = 'shopping';
    console.log(this.value);
    console.log(name);
    console.log(age);
}

bar.prototype.friend = 'kevin';

var bindFoo = bar.bind(foo, 'daisy');

var obj = new bindFoo('18');
// undefined
// daisy
// 18
console.log(obj.habit);
console.log(obj.friend);
// shopping
// kevin

尽管在全局和 foo 中都声明了 value 值，最后依然返回了 undefind，说明绑定的 this 失效了，如果大家了解 new 的模拟实现，就会知道这个时候的 this -> obj。

# 箭头函数中的this指向父级函数的this

箭头函数没有自己的this，箭头函数的this不是调用的时候决定的，而是在定义的时候处在的对象就是它的this。

箭头函数的this看外层的是否有函数，如果有，外层函数的this就是内部箭头函数的this，如果没有，则this是window。

let btn1 = document.getElementById('btn1');
let obj = {
    name: 'kobe',
    age: 39,
    getName: function () {
        btn1.onclick = () => {
            console.log(this);//obj
        };
    }
};
obj.getName();

假如上一层并不存在函数，this指向又是谁？

let btn2 = document.getElementById('btn2');
let obj = {
    name: 'kobe',
    age: 39,
    getName: () => {
        btn2.onclick = () => {
            console.log(this);//window
        };
    }
};
obj.getName();

上例中，虽然存在两个箭头函数，其实this取决于最外层的箭头函数,由于obj是个对象而非函数，所以this指向为Window对象。

由于this在箭头函数中已经按照词法作用域绑定了，所以，用call()或者apply()调用箭头函数时，无法对this进行绑定，即传入的第一个参数被忽略：

var obj = {
    birth: 1990,
    getAge: function (year) {
        var b = this.birth; // 1990
        var fn = (y) => y - this.birth; // this.birth仍是1990
        return fn.call({birth:2000}, year);
    }
};
obj.getAge(2018); // 28

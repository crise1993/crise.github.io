# 前端应该了解的数据结构基础

# 前言

数据结构起源于程序设计，是用计算机来储存、组织数据的方式。数据结构不是使我们学会编程，而是交给我们一种编程的思想，让我们有更好的编程思路。

数据结构是程序员必不可少的知识储备，可以用来有效管理数据对象，解决处理程序的性能问题。

# 概念

数据元素相互之间的关系称为数据结构。算法是指对数据的操作，是解决特定问题求解步骤的描述，在计算机中表现为指令的有限序列，并且每条指令表现为一个或多个操作。

  • 广义的说法：数据结构 = 数据储存 + 算法
  • 狭义的说法：数据结构 = 数据的储存

# 与算法的关系

数据结构与算法是相互依托的关系。计算机解决问题，应该先从具体问题中抽象出一个适当的数据模型，设计出一个可以解决此数据模型的算法：

  • 数据结构 -> 类比于建筑工程中的建筑设计图
  • 算法 -> 类比于工程中的施工设计图

# 分类

逻辑结构：反应数据元素之间的逻辑关系；存储结构：数据结构在计算机中的表示。

线性表中的数据元素之间是一对一的关系。也就是数据元素一个接一个的排列。

# 集合

数据结构中的集合关系就类似于数学中的集合。结构中的数据元素除了同属于一种类型外，别无其它关系。(无逻辑关系)

js中使用Set (opens new window)和Map (opens new window)数据结构进行实现。

# 特点

  • 集合中的数据成员是无序的；
  • 每个数据成员在集合中不能重复，仅且出现一次

# 用途

  • 并集、交集、差集

# 栈

栈是一种被限制操作的线性表。特点是“后入先出”（LIFO - Last In First Out），栈内的元素只能通过列表的一端访问，这一端被称为栈顶。

# JS实现

class Stack {
    constructor() {
        this.dataStore = [];
        this.top = 0;
    }
    push(ele) {
        this.dataStore[this.top++] = ele;
    }
    pop(){
        return this.dataStore[--this.top];
    }
    peek() {
        return this.dataStore[this.top-1];
    }
    length() {
        return this.top;
    }
    clear() {
        this.top = 0;
    }
}

# 使用场景

  • 解决括号匹配检查
  • 浏览器后退、前进，编辑器的undo、redo功能
  • ECS函数调用栈

# 定义

集合、线性表（栈、队列、数组、单向链表）、树、图

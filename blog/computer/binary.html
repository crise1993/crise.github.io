# 一道面试题引发的血案，聊聊js中的二进制数

最近看到一道面试题，

!~-1   // true
!~-0   // false
!~1   // false

如果你不清楚，请一起来学习这篇文章吧。通过本篇文章你能学到：

  • 二进制数表示法
  • 进制转换
  • 二进制的位运算

# 二进制和字节

二进制是计算机储存数据的基本方式。从现代计算机电路来说，只有高电平/低电平两种状态，即为0/1状态，计算机中所有的数据按照具体的编码格式以二进制的形式存储在设备中。

计算机通信和存储的时候都是以0101这样的二进制数据为基础来做处理的，这儿的一个0和1占的地方就叫bit(位)，即一个二进制位。8位组成一个字节，所以字节(Byte)也是长度单位。

位和字节的换算关系如下：

  • 1Byte = 8bit
  • 1KB = 1024B
  • 1MB = 1024KB(2的十次方)

# 二进制数的表示法

以八个二进制位的举例。

# 原码

是最简单的机器数表示法，用最高位表示符号位，其他位存放该数的二进制的绝对值：

  • 第 1 位表示符号位，0 表示正数，1 表示负数。
  • 后 7 位表示真值。
// 1
0000 0001 // 原码
// -1
1000 0001 // 原码

# 反码

原码最大的问题就在于一个数加上它的相反数不等于0，于是反码的设计思想就是冲着解决这一点，既然一个负数是一个正数的相反数，那干脆用一个正数按位取反来表示负数。

  • 正数：是其原码本身不变。
  • 负数：符号位不变，后 7 位的真值按位取反。
0000 0001 // 原码
0000 0001 // 反码
// -1
1000 0001 // 原码
1111 1110 // 反码

# 补码

反码解决了正负相加， 但负负相加结果错误；补码的出现是为了解决负数计算的问题， 不影响正数， 所以正数的补码是本身，负数补码有变化。

  • 正数：是其原码本身不变。
  • 负数：在原码的基础上，符号位不变, 其余各位取反, 最后+1（即反码+1）。
// 1
0000 0001 // 原码
0000 0001 // 反码
0000 0001 // 补码
// -1
1000 0001 // 原码
1111 1110 // 反码
1111 1111 // 补码

补码运算：

// 1 + 2 补码运算
0000 0001 + 0000 0010 = 0000 0011
0000 0011(补码) = 0000 0011(反码) = 0000 0011(原码) = 3

// 1 + (-2) 补码运算
0000 0001 + 1111 1110 = 1111 1111
1111 1111(补码) = 1111 1110(反码) = 1000 0001(原码) = -1

# 进制转换

# 二进制转十进制

js中可以使用parseInt、Number、+（一元运算符）等方法来将二进制数或其他进制数转化为十进制数。

parseInt('110', 2) // 6
Number('0b110') // 6
+'0b110' // 6

# 无符号整数

无符号整数的二进制转化为十进制数，从二进制数的右边第一位起，从右往左，先用二制位置上的数乘以2的相应位数的幂，然后把每一位的乘积相加即可得到二进制数对应的十进制数。

// 二进制：1101001
// 解析：1*2^0+1*2^3+1*2^5+1*2^6 = 105

# 带符号二进制整数

带符号的二进制数转化为十进制数，先观察二进制数最高位是什么数，如果是1，则表示是负数，如果是0则表示是正数，确定符号后再来转化为十进制数。

// 二进制：10000000 00010000
// 解析：-（1*2^4）= -16

# 小数

小数的二进制数转化为十进制数的方法，从左往右，用二进制位数上的数字乘以2的负位数次幂，然后把所有乘积相加即可得。

// 二进制：1.1101
// 解析：1 + 0.8125 = 1.8125
// 整数部分：1*2^0=1
// 小数部分：1*2^(-1) + 1*2^(-2) + 1*2^(-4) = 0.8125

# 十进制转二进制

js中可以使用x.toString(2)方法来将十进制数转化为二进制数。

# 整数

十进制整数转换为二进制整数采用"除2取余，逆序排列"法。

具体做法是：用2整除十进制整数，可以得到一个商和余数；再用2去除商，又会得到一个商和余数，如此进行，直到商为小于1时为止，然后把先得到的余数作为二进制数的低位有效位，后得到的余数作为二进制数的高位有效位，依次排列起来。

// 如：173=（10101101）
// 173/2=86=====余1
// 86/2=43======余0
// 43/2=21=======余1
// 21/2=10=======余1
// 10/2=5========余0
// 5/2=2=========余1
// 2/2=1=========余0
// 1/2=0=========余1

# 小数

十进制小数转换成二进制小数采用"乘2取整，顺序排列"法。

具体做法是：用2乘十进制小数，可以得到积，将积的整数部分取出，再用2乘余下的小数部分，又得到一个积，再将积的整数部分取出，如此进行，直到积中的小数部分为零，此时0或1为二进制的最后一位。或者达到所要求的精度为止。

// 如：0.625=（0.101）
// 0.625*2=1.25======取出整数部分1
// 0.25*2=0.5========取出整数部分0
// 0.5*2=1==========取出整数部分1

// 如：0.7=（0.1 0110 0110...）
// 0.7*2=1.4========取出整数部分1
// 0.4*2=0.8========取出整数部分0
// 0.8*2=1.6========取出整数部分1
// 0.6*2=1.2========取出整数部分1
// 0.2*2=0.4========取出整数部分0
// 0.4*2=0.8========取出整数部分0
// 0.8*2=1.6========取出整数部分1
// 0.6*2=1.2========取出整数部分1
// ...

# 二进制的计算

二进制数据的计算指的是位数据的计算，也就是位运算。

位运算分为以下几种操作：

符号 描述 运算规则 使用场景
& 与 两个位都为1时，结果才为1 判断奇偶 n&1，0为偶，1为奇
| 或 两个位只要有一个为 1 则返回 1，否则返回 0
^ 异或 两个位相同为0，相异为1 交换数值 diff = a^b； b = a^diff; a = b^diff
~ 按位非 0变1，1变0 ～～value 取整数，比 parseInt（value）效率更高
<< 左移 各二进位全部左移若干位，高位丢弃，低位补0 a <<br n = a * 2^n
>> 右移 各二进位全部右移若干位，正数左补0，负数左补1，右边丢弃 a >> n = a / 2^n （a 为正数）
a >> n = （a + 2^32） / 2^n （a 为负数）
>>> 无符号右移 各二进位全部右移若干位，左边补0，右边丢弃 a >> n = a / 2^n （a 为正数）
a >> n = （a + 2^32） / 2^n （a 为负数）

# 位运算的意义

位运算能够高效率的完成数值的计算。因为机器最终是转化为二进制去运算，位运算过程则更为直接。底层使用补码运算加法和移位，实现不仅仅加法的其他运算（减乘除模法等）。

# 关于开头的面试题

再来说说开头的问题，我们可以分解为下面的问题再取反。非运算对于数字的结果相当于改变符号，并对其值的绝对值-1。

~-1
// >>> 0
~0
// >>> -1
~1
// >>> -2

其实可以看出!~x的逻辑就是判断x是否为-1。这让我联想到业务代码的一种其他写法：

!~location.href.search('***')

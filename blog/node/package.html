# Nodejs package包管理指南

在开发Node.js项目时，包管理是至关重要的。它有助于我们管理依赖项，确保项目的可维护性和稳定性。在 Node.js 的包管理系统中，有三种主要的依赖关系：dependencies、devDependencies 和 peerDependencies。

本文一开始将详细解释npm版本的格式和符号，然后围绕dependencies、devDependencies、peerDependencies，深入分析每种依赖的作用、使用场景、安装方式和版本号使用说明。

# npm版本号

# 版本格式

版本格式：主版本号.次版本号.修订号，版本号递增规则如下：

  • 主版本号（Major）：当你做了不兼容的 API 修改
  • 次版本号（MINOR）：当你做了向下兼容的功能性新增
  • 修订号（PATCH）：当你做了向下兼容的问题修正

先行版本号及版本编译信息可以加到“主版本号.次版本号.修订号”的后面，作为延伸。

# 版本符号

版本符号属于语义化版本规范（Semantic Versioning，也称为 SemVer） (opens new window)的一部分。

package.json 中包括版本范围的符号有：

  • ^ 符号表示接受与指定版本相同主版本下的所有次版本和修订版本更新，但不包括新的主版本。
  • ~ 符号表示接受与指定版本号相同的主版本和次版本，但允许修订版本更新。
  • > 符号表示接受比指定版本号更高的任何版本。
  • >= 符号表示接受等于或高于指定版本号的任何版本。
  • < 符号表示接受比指定版本号更低的任何版本。
  • <= 符号表示接受等于或低于指定版本号的任何版本。
  • = 符号用于精确指定特定版本号。

^ 符号指定的版本范围不包括 预发布版本（prerelease）。 ~有助于确保您的项目不会在次要更新时遇到不兼容的更改，但可以获得修复和改进的更新。 >和>=有助于确保您的项目不会安装低于指定版本的包，从而确保您在使用依赖包时至少具有某个最低功能或修复。 <和<=确保您的项目不会安装具有更高版本的包，以避免可能引入的不兼容性或更改。 =确保您的项目使用特定版本的包。

# dependencies（生产依赖）

dependencies 中列出的包是项目在生产环境中所需的依赖项。这些包是项目的核心构建块，它们负责项目的主要功能。

# 使用场景

常见的应用程序或项目的核心依赖应该列在 dependencies 中。

例如：如果你正在构建一个 Web 服务器，Express.js 可能是一个重要的生产依赖，因为它用于处理路由和 HTTP 请求。

# 安装方式

默认情况下，npm install 命令会安装 dependencies 和 devDependencies 中列出的所有依赖项。这包括在生产环境和开发环境中使用的所有依赖。

如果您希望只安装生产环境所需的依赖项，可以使用--production 或 --only=prod选项来排除devDependencies。

# 仅安装生产依赖 (dependencies)
npm install --production
npm install --only=prod

# 版本号使用说明

大多数情况下，dependencies中使用 ^ 或 ~ 符号来指定版本范围比较常见。

但是针对特定项目，比如 项目 和 项目所依赖的npm包 都依赖某个npm包，这时候为了统一npm版本号，避免不同依赖使用不同版本，导致构建输出多份代码的情况，通常便需要使用=或者不加符号来固定版本号。

# devDependencies（开发依赖）

devDependencies 中列出的包是仅用于开发、测试和构建项目的依赖项。它们在生产构建中不会被包含。

# 使用场景

这些依赖用于项目的开发和维护，如测试库、代码规范检查工具、构建工具等。

例如，Mocha 或 Jest 可能是测试的开发依赖，而 Babel 可能是构建工具的开发依赖。

# 安装方式

您可以使用npm install --only=dev 或 npm install --dev命令来安装devDependencies。这将确保只安装开发依赖，而不会包含生产依赖。

# 仅安装开发依赖 (devDependencies)
npm install --dev
npm install --only=dev

# 版本号使用说明

对于 devDependencies，通常使用宽松的版本范围，以便允许在开发期间更新依赖，但不升级到不兼容的版本。这通常使用 ^ 或 ~ 前缀来指定版本范围。

# peerDependencies（对等依赖）

peerDependencies用于指定与您的包一起使用的其他包的版本依赖关系。这有助于确保您的包与特定版本的其他包兼容。

# 使用场景

  • 通常在创建一个公开可用的library时使用（比如npm包）。您可以指定您的包依赖于另一个包的特定版本，以确保其他开发人员在使用您的包时不会遇到不兼容的问题。
  • 也可用于解决包的多版本共存问题，确保只有特定版本的依赖被使用。

# 安装方式

peerDependencies不会自动安装。当其他开发人员安装您的包时，npm 会提醒他们安装与 peerDependencies 中指定的包版本匹配的包。

# 版本号使用说明

在 peerDependencies 中，通常会指定精确的版本号=或允许某个范围的版本。这是因为您希望确保与其他包的特定版本兼容。

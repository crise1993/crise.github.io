# rollup插件详解

@rollup/plugin-commonjs用于将CommonJS模块转换为ES模块 @rollup/plugin-node-resolve 插件来解析第三方Node.js模块 @rollup/plugin-babel插件来将代码转换为ES5

# @rollup/plugin-node-resolve

用来解析第三方模块

比较

// 不配置 @rollup/plugin-node-resolve 插件引入方式
export foo from './foo/index.js'
import bar from './bar/index.js'
// 配置了 @rollup/plugin-node-resolve 插件引入方式
export foo from './foo'
import bar from './bar'

可以看出不配置的话引入路径必须是完整的。

# @rollup/plugin-babel

@rollup/plugin-babel是用于将ES6+代码转换为ES5代码的Rollup插件，它使用Babel进行代码转换。

下面是@rollup/plugin-babel插件的各个参数的用法说明：

  • babelHelpers (default: 'bundled')： 用于指定Babel Helpers的注入方式。有三个可选值：
    • 'bundled'：将Babel Helpers捆绑在每个输出文件中。
    • 'runtime'：使用@babel/runtime模块来共享Helpers，适用于多个文件的情况。
    • 'inline'：将Helpers内联到每个文件中，适用于只有一个文件的情况。
  • exclude (default: undefined)： 排除的文件匹配模式，接受一个正则表达式或者一个字符串数组。插件将不会对匹配到的文件进行转换。
  • extensions (default: ['.js', '.mjs', '.jsx', '.ts', '.tsx'])： 要转换的文件扩展名列表。
  • babelHelpers (default: 'bundled')： 指定Babel Helpers的注入方式，同上面的babelHelpers参数。
  • include (default: undefined)： 包含的文件匹配模式，接受一个正则表达式或者一个字符串数组。插件将只会对匹配到的文件进行转换。
  • presets (default: [])： Babel预设的数组，用于指定需要使用的Babel预设。预设包含了一组Babel插件和配置，用于转换特定版本的- JavaScript。例如，['@babel/preset-env', '@babel/preset-react']可以用来处理ES6+代码和React的JSX语法。
  • babelrc (default: false)： 是否在项目根目录寻找.babelrc文件，并将其配置用于Babel转换。
  • babelHelpers (default: 'bundled')： 同上面的babelHelpers参数。
  • babelrcRoots (default: 'root')： 用于确定寻找.babelrc文件的根目录。可以是字符串或字符串数组。如果设置为字符串'root'，则会在每- 个输入文件的根目录寻找.babelrc文件。如果设置为数组，则会在数组中的每个目录寻找。此选项适用于monorepo等项目结构。
  • compact (default: true)： 是否压缩输出代码，将代码换行去除。
  • comments (default: true)： 是否保留Babel的注释。
  • minified (default: false)： 是否使用Babel Minify插件进行代码压缩。
  • shouldPrintComment (default: undefined)： 一个函数，用于决定是否打印特定的注释。

这些是@rollup/plugin-babel插件的一些常用参数及其用法。你可以根据项目的需求来配置这些参数，以实现对ES6+代码的转换。更多详细信息可以查阅官方文档：rollup-plugin-babel Options。

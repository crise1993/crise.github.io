# V8执行JavaScript原理

# V8中的编译器和解释器

# Ignition

Ignition (opens new window)是一个解释器，作用是将AST转换成ByteCode（字节码），同时会收集TurboFan优化所需要的信息（比如函数参数的类型信息，有了类型才能进行真实的运算）；

函数第一次调用时，Ignition会逐条解释执行生成的ByteCode。

# TurboFan

TurboFan (opens new window)： 是一个编译器，作用是将字节码编译为CPU可以直接执行的机器码。如果一个函数被多次调用，那么就会被标记为热点代码，然后TurboFan将热点字节码转换成优化的机器码，提高代码的执行性能。

但是，机器码实际上也会被还原为ByteCode，这是因为如果后续执行函数的过程中，类型发生了变化（比如sum函数原来执行的是number类型，后来执行变成了string类型），之前优化的机器码并不能正确的处理运算，就会逆向的转换成字节码；

# 1. 生成AST和执行上下文

# AST应用

AST 是非常重要的一种数据结构，编译器或解释器依赖于 AST，而非源代码。

AST 在很多项目中有着广泛的应用，如：

  • Babel 的工作原理是先将 ES6 源码转换为 AST，再将 ES6 语法的 AST 转换为 ES5 语法的 AST，最后利用 ES5 的 AST 生成 JavaScript 源代码；
  • ESLint 将源码转换为 AST，再利用 AST 来检查代码规范化的问题。

# AST生成

AST的生成总共经历3个过程：词法分析 -> 语法分析 -> 语法树（AST）。

举个🌰：

var myName = 'xxxxx'
# 分词/词法分析（tokenize）

将源码拆解成一个个语法上不可再分、最小的单个字符或字符串 token。此外，它将删除空格字符，注释等。最后，整个代码字符串将被拆分为一个token列表。

上面例子经过词法分析输出的token列表结构如下：

[
  Token { type: 'Keyword', value: 'var', start: 0, end: 3 },
  Token { type: 'Identifier', value: 'myName', start: 4, end: 9 },
  Token { type: 'Punctuator', value: '=', start: 10, end: 11 },
  Token { type: 'String', value: 'XXXXX', start: 11, end: 16 },
]
# 语法分析

这个阶段会把词法分析后获取的token列表流转换成 抽象语法树（AST）的形式。 这个阶段会对代码进行语法分析，检查是否有语法错误（SyntaxError）， 如果发现语法错误，会在控制台抛出异常并终止执行。

上面例子经过javascript-ast (opens new window)语法分析生成的AST结构如下：

{
    "type":"Program",
    "start":0,
    "end":16,
    "body":[
        {
            "type":"VariableDeclaration",
            "start":0,
            "end":16,
            "id":{
                "type":"Identifier",
                "start":4,
                "end":9,
                "name":"myName"
            },
            "params":[

            ],
            "body":{
                "type":"VariableDeclarator",
                "start":10,
                "end":16,
                "body":[

                ]
            },
            // ...
        }
    ],
    "sourceType":"script"
}

# 2. 生成字节码

引入字节码是为了解决起初 V8 直接将 AST 转换为机器码而导致移动设备内存占用高的问题。字节码就是介于 AST 和机器码之间的一种代码，与特定类型的机器码无关，字节码需要通过解释器将其转换为机器码后才能执行。

生成 AST 和执行上下文后，解释器 Ignition 会进行词义分析，根据 AST 生成 字节码。

# 3. 执行代码 & JIT

字节码配合解释器和编译器的技术称为 即时编译 (JIT)。

第一次执行的字节码，解释器 Ignition 会逐条解释执行。在执行字节码的过程中，如果发现有热区代码(HotSpot)，即一段代码被重复执行多次，编译器 TurboFan 会在后台把该段热区字节码编译为高效的机器码，提高后续执行效率。
